# Incident Reports

This directory contains comprehensive incident analysis reports generated by the Azure SRE Agent for the Enterprise Banking Portal application.

## Reports

### [INC0010019-analysis.md](./INC0010019-analysis.md)
**Date:** 2026-01-26  
**Title:** Banking Portal - Nginx 503 "no live upstreams" - Cross-host RCA, MTTR, and Recommendations

**Summary:** Critical outage caused by storage layer RAID failure that cascaded through Oracle database and WebLogic application tier, resulting in Nginx 503 errors.

**MTTR:** 49 minutes (1m38s external outage, full stabilization at ~49 minutes)

**Key Sections:**
- Executive summary with MTTR and impact metrics
- Detailed timeline with 8 phases of incident progression
- Root cause analysis with causal chain
- Impact assessment for users and business
- Comprehensive fix recommendations (immediate, short-term, long-term)
- Proactive monitoring with KQL alert queries
- Actionable next steps with 4-phase remediation plan
- Lessons learned and evidence references

## Report Structure

Each incident report follows this structure:

1. **Executive Summary** - High-level overview, MTTR, and root cause
2. **Scope Confirmation** - Affected components and data sources
3. **Detailed Timeline** - Chronological event sequence with phases
4. **Root Cause Analysis** - Technical deep-dive with causal chain
5. **Contributing Factors** - Infrastructure vulnerabilities and gaps
6. **Impact Assessment** - User and business impact
7. **Fix Recommendations** - Immediate, short-term, and long-term actions
8. **Proactive Monitoring** - KQL queries and alert configurations
9. **Open Questions** - Further investigation items
10. **Next Steps** - Phased action plan with owners
11. **Lessons Learned** - What went well/wrong and improvements
12. **Evidence** - Log excerpts and metrics analyzed
13. **Approvals** - Sign-off checklist
